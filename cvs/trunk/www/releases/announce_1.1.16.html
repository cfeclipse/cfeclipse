Announcement: CFEclipse version 1.1.16 released.

9th September, 2004 : 21:00

Installation & update procedures are listed at the bottom of this mail. All current non-CVS users of CFEclipse should hopefully be notified by Eclipse that there is a new release available. If not, follow the update procedures and mention it to us on the CFE lists.

The CFE Development team would like to apologise for the rather large gap in releases. In our defense we claim that real-life has intruded. On the up side we have sorted out around 50 features & bugs that were listed on the Issue Tracker. We've also received more contributions from Mark Drew to whom muchos kudos goes.

What's next

A brief listing of what's hopefully coming up in the next releases of CFE:

  • The beginnings of the parse of your projects
  • Improved CFScript, though this feature is like WinFS for Longhorn...
  • Maybe, if you're lucky, a basic Mach-II plugin

Enhancements Implemented:

  • NEW Context insight for scope variables
  • NEW Component insight available for pre-defined CFCs via user.xml
  • NEW Dictionary reload on Ctrl+F5
  • NEW cfinclude 'template' attribute proposals list directories & files
  • NEW Can change the editors background colour (cheers to Mulcahey, Robert J.)

  • NEW Snip behaviour modified - single character triggers supported, nsert snippet action now searches back and forward for the first character that is not numeric, alpha, _ or -
  • NEW Snip dialog has scroll bars for edit boxes, multi-line snips inserted with same indent as first line
  • NEW F8 saves & refreshes browser if scribble file is already active
  • NEW Now there's only one dialog per snippet for variables & drop-downs for enumerator values
